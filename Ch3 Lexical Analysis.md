# 3.1 The Role of the Lexical Analyzer

![[Pasted image 20260120211915.png]]

lexical analyzer is the part of the compiler that reads source text
may do other things besides identifying lexemes
- strip out comments + whitespace
- correlates error messages generated by compiler with the source program
- if source program uses macro-preprocessor, expansion of macros may be performed by lexical analyzer

lexical analyzers divided into 2 processes:
1. scanning: processes that don't require tokenization of input (deletion of comments, compaction of consecutive whitespace, etc.)
2. lexical analysis: produces tokens from output of scanner

## 3.1.1 Lexical Analysis Versus Parsing

why analysis portion of compiler is separated into lexical analysis + parsing:
1. simplicity of design
2. compiler efficiency
3. compiler portability

## 3.1.2 Tokens, Patterns, and Lexemes

- token: <token name, attribute value>
- pattern: form of lexemes a token may take
- lexeme: instance of a token (identified by lexical analyzer)

## 3.1.3 Attributes for Tokens
- attribute value describes lexeme represented by token
- appropriate attribute value for identifier: pointer to symbol table entry with info on type, first declaration, lexeme, etc.
